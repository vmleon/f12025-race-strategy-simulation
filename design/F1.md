# F1 25 Telemetry UDP Specification

## Overview

F1 25 broadcasts real-time telemetry over UDP. The game sends packets at a configurable rate to a target IP/port. The **UDP Send Rate** is configurable in-game: **10Hz, 20Hz (default), 30Hz, or 60Hz**. Not all packet types are sent every tick — high-frequency packets (Motion, LapData, CarStatus, etc.) are sent every 2 ticks, while lower-priority packets (Session, CarSetups) are sent every ~10 ticks. At the default 20Hz, this produces roughly 80-100 UDP packets/second total across all types.

Each packet starts with a common 29-byte header followed by a type-specific payload. All data is little-endian, packed with no padding. Arrays hold up to 22 car slots (2025 season has 20 active cars).

## Packet Header (29 bytes)

| Offset | Size | Field                   | Type   | Description                                                              |
| ------ | ---- | ----------------------- | ------ | ------------------------------------------------------------------------ |
| 0      | 2    | packetFormat            | uint16 | Game release year (`2025`)                                               |
| 2      | 1    | gameYear                | uint8  | Last two digits (`25`)                                                   |
| 3      | 1    | gameMajorVersion        | uint8  | X.0.0                                                                    |
| 4      | 1    | gameMinorVersion        | uint8  | 0.X.0                                                                    |
| 5      | 1    | packetVersion           | uint8  | Bumped when the layout of this packet type changes in a patch            |
| 6      | 1    | packetId                | uint8  | Packet type (0-15)                                                       |
| 7      | 8    | sessionUID              | uint64 | Random value generated at session start; groups all packets in a session |
| 15     | 4    | sessionTime             | float  | Session elapsed time (seconds)                                           |
| 19     | 4    | frameIdentifier         | uint32 | Simulation tick counter; resets each session                             |
| 23     | 4    | overallFrameIdentifier  | uint32 | Like frameIdentifier but never resets across sessions                    |
| 27     | 1    | playerCarIndex          | uint8  | Index of the player's car in arrays (0-21)                               |
| 28     | 1    | secondaryPlayerCarIndex | uint8  | Splitscreen second player; `255` if not applicable                       |

| ID  | Name                | Size (bytes) | Description                                                |
| --- | ------------------- | ------------ | ---------------------------------------------------------- |

#### Sector time encoding

Sector times and deltas are split into a `uint16` milliseconds part and a `uint8` minutes part. To reconstruct:

```
total_ms = minutesPart * 60000 + msPart
```

A value of 0 for both parts means the sector has not been completed yet on this lap.

### Event (ID 3) - 45 bytes

Single event per packet. `eventStringCode` is a 4-byte ASCII string (not null-terminated).

| Field           | Type    | Notes                                       |
| --------------- | ------- | ------------------------------------------- |
| eventStringCode | char[4] | 4-char event code                           |
| eventDetails    | union   | Interpreted based on event code (see below) |

| RTMT | Retirement           | `vehicleIdx` (uint8), `reason` (uint8, see below)                                                                                               |
| DRSE | DRS Enabled          | None (race control enables DRS for all)                                                                                                         |

| CHQF | Chequered Flag       | None                                                                                                                                            |
| RCWN | Race Winner          | `vehicleIdx` (uint8)                                                                                                                            |

| SPTP | Speed Trap Triggered | `vehicleIdx` (uint8), `speed` (float, km/h), `isOverallFastest`, `isDriverFastest` (uint8), `fastestVehicleIdx` (uint8), `fastestSpeed` (float) |

## Glossary

| Abbreviation | Meaning                                                                                                         |
| ------------ | --------------------------------------------------------------------------------------------------------------- |
| SC           | Safety Car — physical car that leads the field at reduced speed after an incident                               |
| VSC          | Virtual Safety Car — speed limit applied to all cars via delta time, no physical car                            |
| DRS          | Drag Reduction System — rear wing flap that opens on designated straights when within 1 second of the car ahead |
| ERS          | Energy Recovery System — hybrid energy system comprising MGU-K and MGU-H                                        |
| MGU-K        | Motor Generator Unit - Kinetic — recovers energy from braking, deploys to rear axle                             |
| MGU-H        | Motor Generator Unit - Heat — recovers energy from exhaust gases, spins the turbo                               |
| ICE          | Internal Combustion Engine                                                                                      |
| ES           | Energy Store — the ERS battery                                                                                  |
| CE           | Control Electronics — manages power flow between ICE, MGU-K, MGU-H, and ES                                      |
| TC           | Turbocharger                                                                                                    |
| DSQ          | Disqualified                                                                                                    |
| DNF          | Did Not Finish                                                                                                  |
| MFD          | Multi-Function Display — the steering wheel screen showing fuel, ERS, tyre data                                 |
